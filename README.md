# Mekgwe & Co. — Portfolio

A personal portfolio in five chapters, written like a book.

Built with **Next.js 15**, **React 19**, **Tailwind 4**, **Framer Motion**, and **TypeScript**. Hosted on **Vercel**. CI on **GitHub Actions**.

---

## Quick start

> Open this `portfolio/` folder in VS Code (`File → Open Folder…`). The Word CV in the parent directory is intentionally kept outside the project so it stays out of git.

```bash
# 1. Install dependencies
npm install

# 2. Copy the environment template
cp .env.example .env.local

# 3. Run the dev server (Turbopack)
npm run dev
```

Open <http://localhost:3000> — the cover page renders. The book-open animation, chapters, and projects feed land in subsequent commits.

> **Node:** 20 or higher (22 recommended — matches the Vercel runtime).

## VS Code setup

Open this folder in VS Code. For the best experience, install these three extensions when prompted:

| Extension | ID | Why |
| --- | --- | --- |
| Biome | `biomejs.biome` | Lint + format on save (replaces ESLint + Prettier) |
| Tailwind CSS IntelliSense | `bradlc.vscode-tailwindcss` | Class-name autocompletion, palette previews |
| Playwright Test | `ms-playwright.playwright` | Run E2E tests inline |

Add to your user `settings.json`:

```jsonc
{
  "editor.defaultFormatter": "biomejs.biome",
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.organizeImports.biome": "explicit"
  }
}
```

## Editing CV content

`content/cv.json` is the single source of truth. The site re-renders automatically; the downloadable CV PDF (`public/cv.pdf`) is regenerated by the `update-cv` GitHub Action.

To preview a change:

```bash
# 1. Edit content/cv.json
# 2. The dev server hot-reloads. Inspect the result at /chapter/credentials.
# 3. Run tests
npm run test
```

To publish:

```bash
git add content/cv.json
git commit -m "cv: update credentials"
git push
# Vercel deploys; the update-cv workflow regenerates public/cv.pdf and opens a PR.
```

## Environment variables

Copy `.env.example` → `.env.local` and fill in:

| Variable | Required? | Where it's used |
| --- | --- | --- |
| `NEXT_PUBLIC_SITE_URL` | yes | OpenGraph, sitemap, canonical links |
| `NEXT_PUBLIC_GITHUB_USER` | yes | Selected Works chapter (`ipelengmekgwe`) |
| `GITHUB_TOKEN` | optional | Raises GitHub API rate limit 60 → 5000/h |
| `GITHUB_EXCLUDE_OWNERS` | yes | Comma-separated owners to omit from the projects feed |
| `RESEND_API_KEY` | yes (prod) | Powers the contact form on the back cover |
| `CONTACT_TO_EMAIL` | yes | Where contact form submissions land |
| `CONTACT_FROM_EMAIL` | yes | Sender (use `onboarding@resend.dev` until a domain is verified) |

Set the same variables on **Vercel → Project → Settings → Environment Variables**.

## Stack reference

- **Framework:** Next.js 15 with the App Router and Turbopack
- **Styling:** Tailwind CSS 4 (CSS-first config in `globals.css`), CSS variables for the design tokens
- **Type system:** TypeScript 5.7 with `strict` and `noUncheckedIndexedAccess`
- **Animation:** Framer Motion 11 (book-open, page turn, scroll reveal)
- **Forms / email:** Resend for the contact form, Zod for input validation
- **Testing:** Vitest + Testing Library for units, Playwright for E2E
- **Lint / format:** Biome (single binary, replaces ESLint + Prettier)
- **Hosting:** Vercel (auto preview per PR, prod on `main`)
- **Analytics:** Vercel Web Analytics (privacy-friendly, no cookies)

## Deploying to Vercel

Once the repo is pushed to GitHub:

1. <https://vercel.com/new> → **Import** the `ipelengmekgwe/portfolio` repo.
2. Framework preset auto-detects as **Next.js**. Leave the build / output / install commands at the defaults.
3. **Environment Variables** — add every row from the table above. The build is graceful if `RESEND_API_KEY` is missing, but the contact form won't send until it's set.
4. **Deploy.** First build takes ~90 seconds. Subsequent pushes redeploy automatically.

After the first deploy, copy the production URL (e.g. `https://portfolio-xxx.vercel.app`) and update `NEXT_PUBLIC_SITE_URL` in the Vercel env vars to that exact URL — that's what powers OpenGraph/canonical tags. Trigger a redeploy so the change takes effect.

> **Custom favicon:** Vercel serves `src/app/icon.svg` automatically. Drop a higher-res PNG there if you want bitmap variants — Next.js will generate `apple-touch-icon` and the rest from any image you place in that folder.

## Continuous integration

Two GitHub Actions live in `.github/workflows/`:

| Workflow | Trigger | What it does |
| --- | --- | --- |
| `ci.yml` | every push + PR | `biome check`, `tsc --noEmit`, `vitest run`, `next build`. Fails the PR on any error. ~2 min. |
| `lighthouse.yml` | every PR | Waits for the Vercel preview deploy, runs Lighthouse against it, posts scores as a PR comment, fails the check if **performance < 90** or **accessibility < 95**. |

Thresholds live in `lighthouserc.json` — relax or tighten them there. No Vercel tokens are needed; `lighthouse.yml` reads the preview URL from the Vercel comment GitHub creates automatically.

Cancel-in-progress is enabled, so pushing twice in quick succession only spends one CI run's worth of minutes.
